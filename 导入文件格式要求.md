# 导入文件格式要求

本文档详细说明测绘DSP算法集成平台各模块的数据文件导入格式要求。

---

## 通用说明

1. **文件编码**：推荐使用 UTF-8 编码
2. **分隔符**：使用空格或Tab分隔各字段
3. **注释行**：以 `#` 开头的行将被忽略
4. **空行**：空行将被自动跳过
5. **文件格式**：支持 `.txt` 和 `.csv` 文本文件

---

## 模块1：IDW插值

### 文件格式

```
点号 X坐标 Y坐标 高程Z
```

### 格式说明

| 字段 | 类型 | 说明 |
|------|------|------|
| 点号 | 字符串 | 已知点或待插值点的唯一标识 |
| X坐标 | 浮点数 | X方向坐标值（单位：米） |
| Y坐标 | 浮点数 | Y方向坐标值（单位：米） |
| 高程Z | 浮点数 | 已知点的高程值（单位：米） |

### 数据分类规则

- **已知点**：包含4个字段的行（点号、X、Y、Z）
- **待插值点**：仅包含3个字段的行（点号、X、Y）

### 示例文件

```
# 已知采样点（4个字段）
P01 4302.047 3602.652 10.804
P02 4305.768 3598.683 10.855
P03 4310.610 3595.393 10.998
P04 4313.138 3595.086 12.038
P05 4316.843 3594.703 12.818

# 待插值点（3个字段）
Q1 4310 3600
Q2 4330 3600
Q3 4310 3620
Q4 4330 3620
```

### 注意事项

- 已知点至少需要4个才能进行插值
- 点号不能重复
- 坐标值应使用数值格式，避免使用科学计数法

## 模块2：GPS高程拟合

### 文件格式

```
点号 X坐标 Y坐标 高程异常Zeta
```

### 格式说明

| 字段 | 类型 | 说明 |
|------|------|------|
| 点号 | 字符串 | 点的唯一标识 |
| X坐标 | 浮点数 | X方向坐标值（单位：米） |
| Y坐标 | 浮点数 | Y方向坐标值（单位：米） |
| 高程异常Zeta | 浮点数 | 高程异常值（单位：米） |

### 数据分类规则

- **已知点**：高程异常Zeta **不为0** 的数据点
- **待求点**：高程异常Zeta **等于0** 的数据点

### 示例文件

```
# 已知点（Zeta不为0）
1 46.0491 89.6995 -3.9739
2 40.8576 99.6299 -3.5376
3 38.4841 92.1108 -3.7673
4 17.0032 98.8406 -3.3261
5 22.1838 101.9830 -3.2858
6 19.0450 106.2452 -3.0561
7 46.5016 105.1646 -3.3998
8 32.3969 110.7557 -3.0036
9 27.9913 123.9794 -2.4262
10 43.8827 143.2226 -1.9400
11 23.8734 138.8852 -1.7396
12 25.5653 145.7765 -1.5224

# 待求点（Zeta为0）
13 36.3441 123.2001 0
14 34.5588 126.6291 0
15 34.9654 131.8336 0
16 30.5073 129.2705 0
17 38.5731 134.9822 0
18 31.7270 137.7843 0
```

### 注意事项

- 已知点至少需要12个才能进行高程拟合
- 高程异常值为0的点将被视为待求点
- 支持三种拟合模型：平面拟合、二次曲面拟合、四参数曲面拟合

## 模块3：时间系统转换

### 文件格式

```
年 月 日 时 分 秒
```

### 格式说明

| 字段 | 类型 | 取值范围 | 说明 |
|------|------|----------|------|
| 年 | 整数 | 1-9999 | 公历年份 |
| 月 | 整数 | 1-12 | 公历月份 |
| 日 | 整数 | 1-31 | 公历日期 |
| 时 | 整数 | 0-23 | 小时 |
| 分 | 整数 | 0-59 | 分钟 |
| 秒 | 浮点数 | 0-60 | 秒（支持小数） |

### 示例文件

```
# 格式：年 月 日 时 分 秒
2023 5 15 8 30 0
2024 2 28 14 45 30
2025 12 31 23 59 59
2026 3 11 15 30 0
```

### 转换结果

程序将自动计算并输出：
- 儒略日 (JD)
- GPS周 / 周内秒
- 年积日
- 三天打鱼两天晒网状态

## 模块4：多边形面积计算

### 文件格式

```
点号 X坐标 Y坐标
```

### 格式说明

| 字段 | 类型 | 说明 |
|------|------|------|
| 点号 | 字符串 | 多边形顶点的标识 |
| X坐标 | 浮点数 | X方向坐标值（单位：米） |
| Y坐标 | 浮点数 | Y方向坐标值（单位：米） |

### 示例文件

```
# 按顺序排列的多边形顶点
A 1213.045 1040.663
B 1401.734 1143.376
C 1434.466 1331.594
D 1385.811 1549.016
E 1351.343 1757.744
F 1119.667 1629.719
G 1064.570 1360.432
H 1072.567 1170.905
```

### 注意事项

- 顶点数至少需要3个
- 顶点顺序影响面积计算（顺时针/逆时针）
- 程序支持自动排序功能（推荐开启）

## 模块5：坐标转换

### 文件格式

#### XYZ坐标格式

```
点号 X坐标 Y坐标 Z坐标
```

#### BLH坐标格式（Excel导入）

| 列名 | 类型 | 说明 |
|------|------|------|
| B | 浮点数 | 大地纬度（单位：度） |
| L | 浮点数 | 大地经度（单位：度） |
| H | 浮点数 | 大地高（单位：米） |

### 格式说明

| 字段 | 类型 | 说明 |
|------|------|------|
| 点号 | 字符串 | 坐标点的标识 |
| X坐标 | 浮点数 | 空间直角坐标X（单位：米） |
| Y坐标 | 浮点数 | 空间直角坐标Y（单位：米） |
| Z坐标 | 浮点数 | 空间直角坐标Z（单位：米） |

### 示例文件

```
# XYZ空间直角坐标
A -2422455.7116 5377816.7730 2434562.7366
B -2512351.4719 5369568.5389 2432435.1126
C -2478312.5649 5380268.2462 2425129.0941
D -2392240.9396 5397563.0493 2404757.8901
E -2476565.0023 5391602.9203 2392378.8893
F -2523062.3827 5389237.8430 2417327.0448
```

### NEU转换说明

若进行XYZ到NEU转换，需要额外提供参考站坐标：
- 流动站坐标：X、Y、Z
- 参考站坐标：Ref_X、Ref_Y、Ref_Z

## 模块6：滑坡监测

### 文件格式

```
点号 期数
期号1 X1 Y1
期号2 X2 Y2
...
```

### 格式说明

**点号行**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 点号 | 字符串 | 监测点的唯一标识 |
| 期数 | 整数 | 该监测点的观测期数 |

**坐标行**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 期号 | 整数 | 观测期号（如1, 2, 3...） |
| X坐标 | 浮点数 | X方向坐标值（单位：米） |
| Y坐标 | 浮点数 | Y方向坐标值（单位：米） |

### 示例文件

```
# 监测点M01，共4期观测
M01 4
1 492.1373 973.2576
2 492.1377 973.2694
3 492.1421 973.2718
4 492.1436 973.2753

# 监测点M02，共4期观测
M02 4
1 427.5401 481.6885
2 427.5326 481.6854
3 427.5288 481.6845
4 427.5298 481.6887

# 监测点M03，共4期观测
M03 4
1 95.3324 156.7524
2 95.3287 156.7561
3 95.3217 156.7551
4 95.3296 156.7539
```

### 注意事项

- 每个监测点至少需要2期数据才能计算变形速度
- 期号应连续且从1开始
- 坐标值应使用相同的坐标系

## 常见问题

### 1. 文件编码问题

**问题**：导入后显示乱码  
**解决**：确保文件使用UTF-8编码保存

### 2. 分隔符问题

**问题**：数据解析失败  
**解决**：确保使用空格或Tab作为分隔符，避免使用逗号

### 3. 数值格式问题

**问题**：数值解析错误  
**解决**：
- 使用小数点（.）而非逗号（,）
- 避免在数值前后添加空格
- 避免使用千位分隔符

### 4. 点号重复问题

**问题**：计算结果异常  
**解决**：确保每个点号唯一，不重复
